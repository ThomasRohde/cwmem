cwmem is organized under src/cwmem/ with three packages: (1) cli/ — Typer command definitions split by concern: setup.py (init, status, guide + command catalog builder), read.py (get, list, search, related, log, stats), write.py (add, update, tag-add/remove, event-add, entity-add, link, deprecate), graph.py (graph visualization), sync.py (export, import), maintenance.py (build, plan, validate, apply, verify). Commands registered via per-module register(app) functions called from __main__.py. (2) core/ — Business logic: store.py (SQLite CRUD), models.py (50+ Pydantic models), paths.py (directory structure), ids.py (Crockford Base32 + sequential IDs), fingerprints.py (SHA256), events.py (append-only log), fts.py (FTS5), embeddings.py (Model2Vec), hybrid_search.py (RRF fusion), graph.py (entity/edge ops), export.py (deterministic rendering), importer.py (snapshot loading), planner.py (plan generation), validator.py (state validation), safety.py (dry-run/idempotency), locking.py (portalocker), automation.py (future hooks). (3) output/ — envelope.py (JSON envelope builder, exit codes, error types, contextvars for request tracking), json.py (orjson serialization), table.py (planned).
