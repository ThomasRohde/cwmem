cwmem seeds three taxonomy files on init: tags.json, entity-types.json, and relation-types.json under memory/taxonomy/. These provide governed vocabularies for enterprise architecture work. Enforcement is tiered: entity types emit warnings at creation time but don't block; edge relation types are validated but not blocked; 'cwmem verify' runs comprehensive validation across all resources. Taxonomies are committed to git, making vocabulary evolution visible in code review. Default seeds include EA-domain terms (capability, domain, initiative, system, team) and relationship types (depends_on, supersedes, references, owned_by, etc.).
