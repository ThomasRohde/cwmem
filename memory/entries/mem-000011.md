The test suite uses pytest with a subprocess-based CLI integration pattern. The run_cli fixture spawns 'python -m cwmem' in a tmp_path, captures stdout/stderr, and returns CompletedProcess. Helper functions parse_envelope_any_exit() and run_ok() validate JSON envelope structure. Tests are organized by feature: test_entry_crud.py (CRUD + fingerprint tracking), test_events.py (append-only log + lifecycle), test_fts.py (FTS5 indexing), test_hybrid_search.py (lexical + semantic + RRF), test_embeddings.py (Model2Vec pipeline), test_locking.py (lock acquisition + timeout), test_idempotency.py (replay + hash matching), test_dry_run.py (preview without side effects), test_graph_cli.py (entity/edge CRUD + traversal), test_sync_export.py + test_sync_import.py + test_round_trip.py (export/import cycle), test_plan_apply.py (plan workflow), test_verify.py (verification), test_validation_errors.py (input validation + error codes), test_help_cli.py (help text + command discovery), test_guide.py (guide document schema). Phase helpers provide test data builders for progressive feature development.
