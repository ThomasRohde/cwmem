cwmem's retrieval stack has three layers: (1) SQLite FTS5 BM25 lexical search over title, body, and tags; (2) Model2Vec semantic similarity using vendored 384-dim embeddings with cosine distance; (3) optional graph expansion via BFS traversal of edges. Hybrid mode merges lexical and semantic candidate sets using Reciprocal Rank Fusion (RRF) with k=60 (Cormack et al. 2009). Each search hit includes an explanation object with lexical_rank, semantic_rank, rrf_score, and matched_fields. Users can force --lexical-only or --semantic-only modes. Graph expansion adds related resources at configurable depth (1-4). Filters (tag, type, author, date range) apply at the query level before fusion.
