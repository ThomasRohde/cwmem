The cwmem data model has four core resource types stored in SQLite (schema v3): (1) Entries (mem-NNNNNN) — titled knowledge records with body, type, status, author, tags, metadata, provenance, related_ids, and entity_refs. Support CRUD with fingerprint-based concurrency. (2) Events (evt-NNNNNN) — append-only log records with event_type, body, occurred_at, and resource references. Lifecycle events auto-generated on entry create/update. (3) Entities (ent-NNNNNN) — knowledge graph nodes with entity_type, name, description, aliases, and status. (4) Edges (edg-NNNNNN) — directed relationships between any resource pair with relation_type, confidence (0-1), provenance, and is_inferred flag. Self-links are rejected. Duplicate edges (same source/target/relation) are blocked. All resources use Crockford Base32 internal IDs and sequential public IDs.
