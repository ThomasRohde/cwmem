Every cwmem CLI command emits exactly one JSON envelope to stdout with a stable schema: {schema_version, request_id, ok, command, target, result, warnings, errors, metrics}. This machine-first design enables reliable piping to jq, programmatic consumption by AI agents, and consistent error handling. Errors include code, message, retryable flag, suggested_action, and details. Warnings use code+message+details. Exit codes map to error categories: 0=success, 10=validation, 20=auth, 40=conflict, 50=io, 90=internal. Planned but unimplemented: table and markdown output modes via --format flag.
