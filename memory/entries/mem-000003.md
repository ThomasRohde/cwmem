cwmem splits state into runtime (.cwmem/) and tracked (memory/) directories. The sync export produces deterministic, git-friendly artifacts: per-entry markdown with YAML frontmatter, JSONL for all resource types, taxonomy seed JSON, and a SHA256 manifest. Same database state always produces identical bytes. The --check flag detects drift between exported artifacts and current DB state without writing. This enables CI gates: 'cwmem sync export --check' fails if artifacts are stale. The sync import workflow rebuilds the runtime database from tracked artifacts, enabling new clones to bootstrap from committed state.
