The embedding pipeline in core/embeddings.py loads a vendored Model2Vec model from models/model2vec/ (manifest.json + safetensors weights). On 'cwmem build', it reads all entries (title+body concatenated) and events (body), encodes them to 384-dim float32 vectors, normalizes if manifest.normalize=true, and stores as raw bytes in the embeddings table. Fingerprint-based optimization: each embedding row tracks content_fingerprint (SHA256 of text) and model_version; if both match, the row is skipped during rebuild. This makes incremental rebuilds fast — only changed or new entries are re-encoded. The model loads lazily and is cached for the duration of a build session. Query encoding uses the same model for cosine similarity computation during semantic search.
