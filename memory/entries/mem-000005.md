SHA256 fingerprints are computed from canonical JSON of content fields (excluding timestamps). They serve two purposes: (1) Optimistic concurrency control via --expected-fingerprint on update, which rejects stale writes with ERR_CONFLICT_STALE_FINGERPRINT; (2) Rebuild optimization for embeddings and FTS, where content_fingerprint + model_version comparisons skip re-encoding unchanged rows. This avoids expensive full rebuilds on incremental changes. Fingerprints are prefixed with 'sha256:' for self-describing verification.
